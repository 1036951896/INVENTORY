🏛️ Arquitectura de Alto Nivel

🎭 Capas

1. Presentation Layer (Frontend)

  • Responsabilidad: Interfaz de usuario, renderizado, interacción
  • Tecnología: React 19 + TypeScript + Vite
  • Ubicación: /frontend
  • Servidor: Express.js (SPA routing)

2. API Layer (Backend)

  • Responsabilidad: Lógica de negocio, validación, autenticación
  • Tecnología: NestJS + TypeScript + Prisma ORM
  • Ubicación: /backend
  • Servidor: Node.js en Docker

3. Data Layer (Database)

  • Responsabilidad: Persistencia de datos
  • Tecnología: PostgreSQL 14+
  • ORM: Prisma
  • Ubicación: Managed Database en Render

🔧 Backend - NestJS

Arquitectura Modular

Cada módulo sigue el patrón Feature Module:

ProductsModule
├── ProductsController (Rutas HTTP)
├── ProductsService (Lógica negocio)
├── Prisma (Acceso datos)
├── DTOs (Validación entrada)
└── Entities (Modelos)

Módulos Disponibles

  • Products Module: CRUD de productos
  • Orders Module: Gestión de órdenes
  • Auth Module: Autenticación JWT
  • Users Module: Gestión de usuarios
  • Categories Module: Categorías de productos
  • Offers Module: Ofertas especiales

Autenticación JWT

// auth.service.ts
async login(email: string, password: string) {
  const user = await this.usersService.findByEmail(email);
  
  if (!user || !bcrypt.compareSync(password, user.password)) {
    throw new UnauthorizedException();
  }
  
  const token = this.jwtService.sign({
    sub: user.id,
    email: user.email,
    role: user.role
  });
  
  return { access_token: token, user };
}

📊 Flujo de Datos - Compra de Producto

1. Usuario hace clic en "Agregar al Carrito"
   ↓
2. CartContext.addToCart(product, quantity)
   ↓
3. Notificación de éxito (Sonner)
   ↓
4. Usuario va a Checkout
   ↓
5. Confirma orden → POST /api/v1/orders
   ↓ BACKEND
6. OrdersService.create()
   ↓
7. Valida stock y crea Order en DB
   ↓
8. Retorna { orderId, numero, total }
   ↓ FRONTEND
9. CartContext.clearCart()
   ↓
10. Redirige a /confirmacion/:orderId
    ↓
11. Muestra confirmación con número orden

🎯 Patrones de Diseño

  • Module Pattern: NestJS módulos independientes
  • Service Layer: Controllers, Services, Prisma
  • Context API: Estado global sin props drilling
  • Custom Hooks: React hooks personalizados
  • Guard Pattern: Protección de rutas
  • DTO Pattern: Separación entrada/salida
  • Strategy Pattern: JWT auth strategies

🔒 Seguridad

Frontend

  • Token JWT en localStorage
  • Headers con Authorization Bearer
  • Rutas protegidas con autenticación
  • Validación TypeScript

Backend

  • Contraseñas con bcrypt (10 rounds)
  • Validación de entrada con DTOs
  • Guards JWT en rutas protegidas
  • Roles (admin/cliente) con autorización
  • CORS configurado correctamente
  • Variables de entorno para secretos

🚀 Deploy - Render.com

Configuración render.yaml

Frontend:
  - Build: npm install && npm run build
  - Start: npm start (Express server)
  - Runtime: Node.js

Backend:
  - Build: Docker image
  - Expose: Puerto 3000
  - Runtime: Node.js in Docker

URLs en Producción

  • Frontend: https://inventory-2-sewi.onrender.com
  • Backend: https://inventory-1-jkh2.onrender.com
  • API Base: https://inventory-1-jkh2.onrender.com/api/v1

Proceso Deploy Automático

1. Push a GitHub (main branch)
       ↓
2. Render webhook se activa
       ↓
3. Frontend build: npm run build → dist/
       ↓
4. Express server sirve dist/
       ↓
5. Backend build: Docker image
       ↓
6. Backend run: npm run start:prod
       ↓
7. Prisma: Aplica migraciones
       ↓
8. Servicios online en producción
